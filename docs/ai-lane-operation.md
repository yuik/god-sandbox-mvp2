# AIレーン運用ルール

## 概要

本ドキュメントは、GodSandbox プロジェクトにおける AI エージェントの担当レーンと運用ルールを明文化したものです。
PBI 着手前に必ず確認してください。

## AIレーン定義

| エージェント | 担当レーン | 主な責務 |
|---|---|---|
| **Copilot** | 実装レーン A | GitHub Copilot coding agent。PBI の実装・PR 作成・ドキュメント追加を担当する。 |
| **Codex** | 実装レーン B | OpenAI Codex。ローカルでのコード生成・リファクタリング支援を担当する。 |
| **Claude（実装役）** | 実装レーン C | Claude coding agent。Copilot / Codex が扱えない複雑な実装タスクを担当する。 |
| **Claude（監査役）** | 監査レーン | Claude review agent。PR レビュー・品質監査・セキュリティチェックを担当する。実装は行わない。 |

## レーン混線防止ルール

1. **実装役と監査役を兼任しない。**
   同一エージェントが同一 PR の実装と承認を行ってはならない。

2. **PR 作成者と Approver は別アカウントにする。**
   実装レーン（Copilot / Codex / Claude 実装役）が作成した PR は、監査レーン（Claude 監査役）または人間のオーナーが承認する。
   実装したエージェント自身が Approve・マージしてはならない。

3. **担当外レーンの作業ファイルに触れない。**
   別のエージェントが着手中の PBI ファイルを上書き・削除してはならない。
   着手中ブランチのファイルが不明な場合は、オーナーに確認してから作業する。

4. **保護パスの変更は manual-review-required ルートを使う。**
   `.github/**`、`AGENTS.md`、`docs/product/**`、`docs/architecture/**`、`src/core/**`、`src/persistence/**` など AGENTS.md に列挙された保護パスへの変更は、必ず `manual-review-required` ラベルを付与して人間のレビューを待つ。

## PR ラベルルール

| ラベル | 使用条件 |
|---|---|
| `agent-routine` | 小規模・可逆・日常的な変更。コード・設定・依存関係・保護パスへの変更を含まない場合。 |
| `manual-review-required` | 破壊的・不可逆・ポリシー変更・セキュリティ影響・保護パス変更を含む場合。 |

どちらか判断に迷う場合は `manual-review-required` を選択する。

## 運用フロー（簡略）

```text
PBI 着手
  |
  +- ブランチ作成（main への直接 push 禁止）
  |
  +- 実装レーン（Copilot / Codex / Claude 実装役）が PR 作成
  |    +- 適切なラベルを付与
  |
  +- 監査レーン（Claude 監査役）または人間オーナーがレビュー・Approve
  |    +- 実装エージェント自身は Approve 不可
  |
  +- マージ
```

## 注意事項

- このドキュメント自体（`docs/ai-lane-operation.md`）は `agent-routine` 対象ファイルです。内容変更は軽量レビューで可です。
- レーン定義やルール自体を変更する場合は、オーナーに確認してから更新してください。
- 本ドキュメントは最小運用メモであり、詳細なワークフローは今後の PBI で拡張できます。
