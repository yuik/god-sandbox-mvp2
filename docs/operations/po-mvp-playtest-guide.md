# PO MVP 手動テストガイド

状態: PO 検証用アーティファクト（仕様正本ではありません）

このガイドは GodSandbox MVP の受け入れ条件を PO が手動で確認するための手順書です。
技術的な前提知識は必要ありません。

正本仕様は以下を参照してください：
- `docs/product/faith-system-spec.md`
- `docs/product/observed-dialogue-spec.md`
- `docs/product/passport-outside-world-spec.md`

---

## 起動前チェック

ブラウザを開く前に確認：

1. `npm run build` が成功していること
2. `npm run test:domain` と `npm run test:ai` が全て `ok` になっていること
3. 開発サーバーが起動していること（通常 `npm run dev`）

---

## Check 1: 箱庭画面が開く

目的：
ゲームの起点となる箱庭画面が正常に表示されることを確認する。

手順：
1. ブラウザで開発サーバーのURLを開く
2. 箱庭画面（キャラクターが表示されるメイン画面）が表示されることを確認する
3. キャラクターが最低 1 人以上表示されることを確認する

期待結果：
- 箱庭画面が読み込まれる
- キャラクターのスプライト（または名前）が表示される
- エラーメッセージや白い画面が出ない

NG：
- 画面が真っ白または真っ黒のまま
- コンソールに未処理エラーが出続ける

---

## Check 2: 観察型発話（Observed Dialogue）の確認

目的：
キャラクターが「生活音のような会話」として発話するかを確認する。
ユーザーへの直接呼びかけではなく、世界に向けた発話であることが重要。

手順：
1. 箱庭画面で数秒〜数十秒待つ（idle_timer トリガー）
2. 画面上にセリフが出ることを確認する
3. セリフが 3〜5 秒程度で消えることを確認する
4. 介入（後述の Check 3）を行い、介入後に発話が出ることも確認する

期待結果：
- セリフが吹き出しや字幕形式で表示される
- セリフが自然に消える
- 同時に表示されるのは最大 2 件まで
- 発話内容が箱庭の世界観に合っており、不自然な単語（内部値・技術語）が出ていない（現行 MVP は汎用短文 fixture のため、キャラ固有の口調強化は次フェーズ）

NG：
- 「あなた」「プレイヤー」「神様」など、ユーザーへの直接呼びかけが含まれる
- 「信仰」「信仰度」「faith」「faithBand」などのゲーム内部用語が含まれる
- 「画面」「ボタン」「セーブ」「ステータス」などの UI 用語が含まれる
- 「信仰度:40」「好感度:85」のような数値付きのゲーム指標が含まれる
- セリフが消えず永続表示される

---

## Check 3: 介入の確認

目的：
神様として「見守る」「助ける」「試練」の 3 種類の介入が機能することを確認する。

手順：
1. 箱庭画面でアクティブなキャラクターを選択または介入ボタンを押す
2. 「見守る（watch）」「助ける（help）」「試練（trial）」のいずれかを選択する
3. 介入結果が画面に表示されることを確認する
4. GodPoints（神様ポイント）が消費されることを確認する

期待結果：
- 介入の結果（成功・失敗）が表示される
- GodPoints が減少する
- 介入後にキャラクターが何らかの反応を示す（発話・状態変化）

NG：
- 介入しても何も変わらない
- GodPoints が減らない
- エラーダイアログが出る

---

## Check 4: 信仰値（faith）が UI に数値として出ないことの確認

目的：
信仰度の「数値」がプレイヤーに直接見えないことを確認する。
「距離感の表現」として出るのは正常。数値そのものはゲーム内部にのみ存在する。

手順：
1. 箱庭画面全体を目視で確認する
2. キャラクターのステータスや介入結果の表示を確認する
3. 「faith」「faithBand」「信仰度」「信仰」などの文字を探す
4. 0〜100 の数値で「信仰」に関係する数字が出ていないか確認する

期待結果：
- 「信仰度: 73」のような数値表示がない
- Passport の JSON ビューアを開いた場合は `currentFaith` が数値として入っているのは正常
- 「まだ距離がある」「気配を感じている」などの日本語表現は OK

NG：
- 画面のどこかに「信仰: 73」「faith: 30」のような数値が表示される
- 介入結果に信仰度の増減数値が直接表示される（例：「信仰度 +5」）

---

## Check 5: 対話ハンドオフ（Dialogue Handoff）の確認

目的：
GodSandbox が外部 AI（ChatGPT など）向けの依頼文を正しく生成し、コピーできることを確認する。
この機能は外部 AI に自動送信は**しない**。PO がコピーして手動で貼り付ける。

手順：
1. 「発話プレビュー」または「外部 LLM 経由の発話候補」のセクションを開く
2. 「依頼文をコピー」ボタンを押す
3. コピーされた文章をメモ帳などに貼り付けて内容を確認する
4. 確認ポイント：
   - キャラクター名が含まれている（「Ryo」「鈴」など）
   - 「JSON 配列で返してください」のような形式指定がある
   - 「6 to 10 items」のような件数指定がある
   - 信仰度の数値（例：「faith: 73」）が含まれていない
   - 「divinePerceptionBand」または「距離感」の記述的表現が含まれている

期待結果：
- 依頼文がクリップボードにコピーされる
- 依頼文を ChatGPT に貼り付けるとキャラクターらしい発話候補が返ってくる
- 依頼文に「信仰度」の数値が露出していない

NG：
- ボタンを押しても何もコピーされない
- コピーした文章に「currentFaith」「faith: XX」のような生の数値が含まれる
- 「信仰段階」「faithBand」がそのままのキーワードとして露出する

---

## Check 6: Passport 発行の確認

目的：
キャラクターの記憶・関係性・信仰の歩みを「パスポート」として発行できることを確認する。

手順：
1. 箱庭画面でキャラクターを選択し、スナップショット（記録）を取る
2. スナップショットから Passport の発行を試みる
3. 初回は「Passport 確認画面」が表示されることを確認する
4. 「含まれるもの」「含まれないもの」のリストが表示されることを確認する
5. キャンセルした場合にゲーム状態が変わっていないことを確認する
6. 確認後に Passport が発行されることを確認する

期待結果：
- 初回確認画面が表示される
- キャンセルでゲームが変化しない
- 2 回目以降は確認画面をスキップ
- Passport JSON が生成される

NG：
- 確認画面が出ない（初回なのにいきなり発行）
- キャンセルでもゲーム状態が変わる
- Passport が生成されない

---

## Check 7: Passport の外部 AI テキストの確認

目的：
Passport の `externalAiPromptBlock`（外部 AI 向け指示文）に信仰度の数値が含まれないことを確認する。

手順：
1. Passport を発行する（Check 6 の後）
2. Passport の JSON ビューアを開く（「詳細を確認する」などのボタン）
3. `godRelationship` セクションを確認する
   - `currentFaith` が数値として存在している → **正常**（JSON 内部には入っていてよい）
   - `faithBand` が `"senses_presence"` 等の文字列で入っている → **正常**
4. `externalAiPromptBlock` セクションを確認する
   - `systemPrompt` の中に「faith: 73」などの数値が含まれていないことを確認
   - 「まだ距離がある」「気配を感じている」のような日本語表現になっている → **正常**

期待結果：
- `externalAiPromptBlock.systemPrompt` に信仰度の数値が出てこない
- `externalAiPromptBlock.systemPrompt` が 50 文字以上の有意な文章になっている
- `externalAiPromptBlock.systemPrompt` をコピーして ChatGPT に貼ると、キャラクターとして振る舞う

NG：
- `systemPrompt` に「currentFaith」「faith: 73」等が含まれる
- `systemPrompt` が空または 10 文字以下
- JSON ビューアに「API」「プロンプト」「トークン」「モデル」のような技術語が UI テキストとして露出する

---

## NG 例まとめ

以下が箱庭 UI や外部 AI 向けテキストに出た場合は問題です：

| カテゴリ | NG となる文字列・パターン |
|---|---|
| 信仰数値 | `faith: 30`、`currentFaith: 73`、`信仰度: 40` |
| ゲーム内部キー | `faithBand`、`divinePerceptionBand`（UI表示として）、`FaithChangeTrigger` |
| UI メタ語 | `画面`、`ボタン`、`セーブ`、`ステータス`、`UI` |
| 直接呼びかけ | `あなた`、`プレイヤー`、`神様`（箱庭内発話で） |
| 技術語（Passport UI） | `API`、`プロンプト`、`トークン`、`モデル`、`自動的に生成`、`システムプロンプト` |
| 五行内部値 | `wood`、`fire`、`earth`、`metal`、`water` |

## merge 後の回帰チェック

新しい PR が main に merge された際に最低限確認すること：

```bash
npm run typecheck        # 型エラーなし
npm run test:domain      # 28 件以上の domain テスト ok
npm run test:ai          # 37 件以上の AI テスト ok
npm run build            # ビルド成功
```

目視確認項目：
- [ ] 箱庭画面が開く
- [ ] 観察型発話が出る（idle_timer 待機）
- [ ] 介入が機能する
- [ ] 信仰数値が UI に露出していない
- [ ] Passport が発行できる
- [ ] externalAiPromptBlock に信仰数値が出ていない
