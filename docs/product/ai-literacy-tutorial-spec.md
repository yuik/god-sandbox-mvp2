# AI リテラシー導線仕様書（AI Literacy Tutorial）

状態: 管理対象の正本ドキュメント

この文書は GodSandbox において「AIが不安な創作ユーザー」が段階的に AI を理解していく導線を設計する。
実装前に `docs/product/core-experience-spec.md` を必ず読むこと。

---

## 1. 設計方針

**ターゲット：** 「AIには興味あるけど怖い」「ChatGPT は使ったことあるけど自分のキャラに使うのは不安」な創作ユーザー。

**MVP 重点フェーズ：** Passport 発行前後（本仕様では Level 1 と Level 3）。プレイヤーが「外に連れ出したい」と思ったタイミングで AI の意味を学ぶ。

**基本原則：**
- Level 0 が常に成立すること（AI なしでも遊べる保証）
- ユーザーが望まなければ Level 0 から進まなくてよい
- AI に「お任せ」しているのではなく「確認している」感覚を保つ
- 何が AI に送られるかを事前に見せる

---

## 2. レベル定義

### Level 0：AIなしで完結

**目的：** AI が全く不要であることを最初に保証する。

**実現する体験：**
- キャラクター作成（5 項目入力）
- 箱庭観察（10 分間自律生活）
- 介入（watch / help / trial）
- snapshot 記録

**UI 要件：**
- 「AIなしでも完全に遊べます」という明示的なメッセージを初回起動時に表示
- AI 関連のボタン・機能は非表示または「任意」ラベルを付与
- AI エラー時にゲームが止まらない（フォールバック必須）

**テキスト例：**
```
「この子は、AIがなくても生きています。
 箱庭で見守るだけで、十分です。」
```

---

### Level 1：何が AI に渡るかを確認する

**目的：** Passport 発行ボタンを押す前に「これを AI に渡していいですか？」を見せる。

**トリガー：** プレイヤーが初めて「Passport を発行する」ボタンをタップしたとき。

**表示内容（確認画面）：**

```
【Passport に含まれる情報の確認】

以下の情報が Passport JSON に含まれます。

この時点では、外部 AI には送られません。
あなたが後でコピーして貼り付けた場合だけ、外部 AI がこの情報を読みます。

✓ {name} の名前・性格の要約
✓ {name} の話し方（口調・禁止事項）
✓ 箱庭での出来事の要約（数字なし、自然言語）
✓ 神との関係の段階と解釈
✓ 発話例

✗ 含まれないもの
  - 陰陽五行の内部値
  - あなたのアカウント情報
  - GodSandbox 固有の計算パラメータ

※ 信仰度の詳細な数値は、JSON の中身を開いた場合だけ確認できます。
※ 通常の確認画面では強調表示しません。

[確認して続ける] [キャンセル]
```

**技術要件：**
- 確認前に API 呼び出しは行わない
- キャンセルしてもゲーム状態は変わらない
- 確認済みフラグを LocalStorage に保存し、2 回目以降はスキップ可（設定で変更可能）

---

### Level 2：生成候補を確認して採用する

**目的：** AI が生成した内容（発話例・narrative）をそのまま採用せず、「候補として確認」するフローを体験する。

**対象：** AI 生成のスプライト画像・発話例・Passport narrative

**フロー：**

```
AI 生成リクエスト（外部ツール、Codex など）
         ↓
  incoming フォルダへ保存
         ↓
  [候補を確認] ボタンが UI に表示
         ↓
  ユーザーが確認画面で内容を確認
         ↓
  [採用する] → confirm → adopt パイプライン
  [やり直し] → incoming 削除
  [後で決める] → pending 状態のまま
```

**確認画面の UI 要件：**
- 候補と「元の設定」を並べて表示する
- 採用ボタンを目立たせすぎない（無意識のワンクリックを防ぐ）
- 「どこが AI 生成か」を明示するラベル（「AI が提案」バッジ）

**フォールバック：**
- 候補が採用されなくてもゲームは進む
- 発話例がなければデフォルト発話セットを使う
- 画像候補がなければデフォルトスプライトを使う

---

### Level 3：Passport JSON を学ぶ

**目的：** Passport JSON の中身を「読める」ようにする。外部 AI に貼るときの意味を理解する。

**トリガー：** Passport 発行完了後、「JSON の中身を見る」ボタンをタップしたとき（任意操作。開かなくてもゲームは進む）。

**UI：インタラクティブ JSON ビューア**

各フィールドにホバー（PC）またはタップ（モバイル）すると説明が表示される：

```json
{
  "character": {           // ← 「{name} の基本情報」
    "name": "Garan",
    "personalitySummary": "落ち着いた観察者…"  // ← 「あなたが設定した性格」
  },
  "lifeMemory": {          // ← 「箱庭での記憶。AI に渡される履歴」
    "memorySummary": "水汲み場での口論を…"
  },
  "godRelationship": {     // ← 「神との関係。信仰度バンドで決まる」
    "faithBand": "senses_presence",  // ← 「数値は非表示。バンド名のみ」
    "interpretationOfGod": "…"
  },
  "externalAiPromptBlock": {  // ← 「外部 AI への指示文。これをコピーして使う」
    "systemPrompt": "…"
  }
}
```

**学習コンテキストパネル（任意表示）：**

```
Q. 信仰度の数値はここに入らないの？
A. 信仰度はバンド名（senses_presence など）として入っています。
   詳細な数値（currentFaith）は Passport JSON の内部データにも含まれますが、
   通常の確認画面では強調表示しません。
   JSONビューアの詳細表示（任意）を開くと確認できます。

Q. これをどこに貼ればいいの？
A. ChatGPT や Claude で新しい会話を開いて、
   最初のメッセージとして貼り付けると、
   {name} として話してくれます。
```

## 3. MVP 実装スコープ

MVP では「Passport 発行前後の体験」を重点実装する。

| 実装 | 優先度 | 備考 |
|---|---|---|
| Level 0（AI なし動作保証） | **必須**（前提条件） | ゲーム全体を通じて常に成立させる |
| Level 1（Passport 前の確認画面） | **MVP 必須** | Passport ボタン初回タップ時に表示 |
| Level 3（JSON ビューア） | **MVP 必須** | Passport 発行後に「JSON を見る」から起動 |
| Level 2（候補確認フロー） | 後回し可 | Codex 生成運用が確立してから実装 |

**MVP の実装優先順：** Level 0 → Level 1 → Level 3 → Level 2

Level 2 は機能的には独立しており、Level 1/3 が完成していなくても着手できる。
ただし Level 0 はすべてのレベルの前提であり、最初に確認すること。

## 4. ユーザー状態の管理

```ts
type AiLiteracyProgress = {
  level0Completed: boolean;   // 常に true（AIなし動作保証）
  passportConfirmSeen: boolean;   // Level 1 確認画面を見たか
  jsonViewerUsed: boolean;        // Level 3 JSON ビューアを使ったか
  candidateAdopted: boolean;      // Level 2 候補採用を体験したか
};
```

LocalStorage または character の meta に保存。サーバー送信不要。

## 5. テキスト設計（ライティングガイドライン）

### 禁止ワード

以下をチュートリアルテキストに使ってはならない：

```
✗ 「AI が考えてくれます」（主体がユーザーであるべき）
✗ 「自動的に生成されます」（無意識の委任を印象づける）
✗ 「プロンプト」（技術者以外に馴染まない）
✗ 「API」（UI には出さない）
✗ 「トークン」「モデル」（不要な技術語）
```

### 推奨ワード

```
✓ 「あなたが確認して採用します」
✓ 「候補として提案されます」
✓ 「{name} らしいかどうか、あなたが判断してください」
✓ 「外部 AI に渡す情報」（APIとは言わない）
✓ 「コピーして貼り付ける」（具体的操作）
```

## 6. エラー・失敗時の設計

| 状況 | 表示 | ゲームへの影響 |
|---|---|---|
| Passport 生成中にタイムアウト | 「生成に時間がかかっています。後で試してください。」 | なし。再試行ボタンを表示 |
| 候補生成が失敗 | 「今回は候補を生成できませんでした。」 | なし。デフォルトを使用 |
| JSON ビューアが開けない | 「JSON の表示に問題があります。」 | なし。Raw JSON をテキストでコピー可能に |
| AI 接続エラー | メッセージなし（Level 0 のまま継続） | なし |

---

## 7. テスト要件

1. AI なし状態でキャラクター作成から snapshot まで完走できる（Level 0）
2. Passport ボタンタップ時に確認画面が表示され、キャンセルでゲーム状態が変わらない（Level 1）
3. 確認画面に「信仰度の数値」「陰陽五行の内部値」が表示されない（Level 1）
4. 確認画面に「含まれないもの」リストが表示される（Level 1）
5. JSON ビューアで全フィールドのラベルが表示される（Level 3）
6. `faithBand` の値が日本語説明付きで表示される（Level 3）
7. チュートリアルテキストに禁止ワード（「API」「プロンプト」「自動的に生成」）が含まれない
8. Level 2 フォールバック：候補が存在しなくてもゲームが止まらない
9. `passportConfirmSeen` フラグが LocalStorage に保存され、2 回目の Passport 発行で確認画面がスキップされる
10. タイムアウト時にリトライボタンが表示され、ゲームが進行可能なまま残る
