2. 現在の安全境界

絶対的な境界

L1 — State Authority

AI（LLM）はゲーム状態を直接変更しない。テキスト提案のみ。状態更新はゲームロジックが決定する。

L6 — Safety Guardrail

LLM 出力は必ず output_guard を通過する。「あなた」「神様」「プレイヤー」等の禁止語は自動ブロック。

L5 — Memory / Lore Context

LLM に渡すコンテキストに信仰度の数値・五行内部値を含めない。記述的なバンド（文字列）のみ。

GodSandbox はゲーム本体から外部 AI API を直接呼び出しません。
LLM との接続は PO が手動でコピー＆ペーストする設計です。

5. Passport Boundary

Passport JSON の構造

godRelationship（内部）

currentFaith: 73（数値として存在）  |  faithBand: "believes"

✓ Passport JSON の内部には currentFaith が含まれる。これは正常。

externalAiPromptBlock（外部 AI 向け）

systemPrompt: 「Ryo はこんな子です…気配を感じている」（日本語記述のみ）

✓ 信仰度の数値 73 はここに出てこない。自動テストで保証済み。

Passport 確認画面のルール

  • 初回のみ「含まれるもの／含まれないもの」の確認画面を表示
  • キャンセルでゲーム状態は変わらない（自動テストで保証済み）
  • 禁止ワード（API、プロンプト、トークン、モデル、自動的に生成、システムプロンプト）は UI テキストに使わない

  • 数秒で自動的に消える

8. PO 手動テストチェックリスト

自動テスト（CI）でカバーできない体験品質の確認項目です。 詳細な手順は docs/operations/po-mvp-playtest-guide.md を参照してください。

  • 箱庭画面が開き、キャラクターが表示される
  • 待機中に観察型発話が出る（idle_timer）
  • 発話が 3〜5 秒で消える
  • 発話に「あなた」「プレイヤー」「神様」が含まれない
  • 発話に「信仰度:XX」「faith: XX」等の数値が含まれない
  • 「見守る」「助ける」「試練」の介入が機能する
  • 介入後に発話または状態変化が起きる
  • 信仰度の数値が箱庭 UI のどこにも表示されない
  • 依頼文コピーボタンが機能する
  • コピーした依頼文に信仰数値が含まれない
  • Passport が発行できる
  • Passport 確認画面（初回）が表示される
  • 確認キャンセルでゲーム状態が変わらない
  • Passport の externalAiPromptBlock に信仰数値が含まれない
  • Passport の systemPrompt をコピーして ChatGPT に貼ると、キャラクターらしく振る舞う
