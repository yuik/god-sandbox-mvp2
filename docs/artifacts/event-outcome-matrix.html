祠の油揚げ shrine-fox-offering ACCUMULATIVE
situationTags
shrine offering accumulative chain-trigger
participants
1名（不定期発生）
structuredPayload
offeringCollected: boolean（success 時 true）
offeringCount: number（セッション通算の成功回数、UIには数値として表示しない）
outcome, judgement, outcomeSummary
principleProfile
dominantPhase: wood / polarity: yin / principleRole: circulate
推奨介入
watch: 誰が置いたか見守る
help: 油揚げを守る
trial: 持ち去られるか試す
蓄積ルール
success ごとに offeringCount +1。
offeringCount >= 3 になったとき chained event: fox-shrine-visitor を発生。
failure では count は増えない。
count の数値は UI に強調表示しない。「祠に気配が濃くなっている」等の自然言語で表す。
Sprint 9 scope
Sprint 9 では template 追加・success/failure summary・offeringCollected の書き込みまで。
fox-shrine-visitor chained event の発生ロジックと UI は次 PBI に委譲。
ただし型・コードコメントで将来の chain 接続を示す。
event_started
dialogue intent
祠のそばに、また油揚げが置いてある。
event_resolved
dialogue intent
成功（count < 3）: 「祠の気配が少し濃くなった」
成功（count >= 3）: 「祠の前で、何かが動いた気がする」
失敗: 「油揚げはいつのまにか消えていた」
outcome
SUCCESS
「祠の気配が少し濃くなった」
  • structuredPayload.offeringCollected = true
  • offeringCount +1
  • harmony +1
  • insight +1
  • count >= 3 で chained event: fox-shrine-visitor
FAILURE
「油揚げはいつのまにか消えていた」
  • structuredPayload.offeringCollected = false
  • offeringCount 変化なし
  • stress +1
chained event
fox-shrine-visitor（templateId）
Sprint 9 では予約のみ。template 定義と chain 条件のコメントを残す。実装は次 PBI。
Dice Resolution Summary
formula
total = rollD20(seed) + modifier
threshold default: 11（legendary-big-fish のみ 13）
total >= threshold → success
modifier rules
条件 modifier
watch かつ insight >= 60+2
help かつ (empathy >= 60 または harmony >= 60)+1
trial かつ courage >= 60+2
stress >= 70（介入種別問わず）-1

Could（Sprint 9 では任意）: principleProfile と character の implicit phase が合う場合に +1

不採用
fumble / critical / greatSuccess は Sprint 9 外。成功/失敗の2択のみ。
